## 🤝 文档贡献指南

### 更新版本文档流程

1. **发布新版本时**:
   ```bash
   # 1. 创建版本说明文档 (使用下一个可用编号)
   docs/10-版本更新/XX-vX.X版本说明.md
   
   # 2. 更新 CHANGELOG
   docs/10-版本更新/00-CHANGELOG.md
   
   # 3. 更新本 README 索引
   docs/10-版本更新/01-README.md
   
   # 4. Git 提交
   git add docs/10-版本更新/
   git commit -m "docs: 添加 vX.X 版本文档"
   ```

2. **文档格式要求**:
   - 使用 Markdown 格式
   - 包含清晰的目录结构
   - 添加适当的 emoji 图标
   - 提供代码示例
   - 标注文件位置

3. **版本号规则**:
   - 主版本号 (X): 重大架构变更
   - 次版本号 (Y): 新功能添加
   - 修订号 (Z): Bug 修复

### 文档归档规则

当版本文档数量超过 10 个时:
1. 保留最近 8-10 个主要版本的文档
2. 将较早版本的详细文档移至 `99-归档/`
3. 在 CHANGELOG 中保留所有版本的简要记录
