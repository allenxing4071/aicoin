## 🎯 功能演进

### AI能力
```
v1.0: 简单规则 AI
  ↓
v2.0: DeepSeek 单引擎
  ↓
v2.1: Qwen情报官 + DeepSeek交易官 双引擎
```

### 数据存储
```
v1.0: SQLite 单数据库
  ↓
v2.0: Redis + PostgreSQL 双存储
  ↓
v2.1: Redis + PostgreSQL + Qdrant 三存储
```

### 权限系统
```
v1.0: 固定权限
  ↓
v2.0: L0-L5 动态权限
  ↓

```

```
v1.0: 无
  ↓
v2.0: 无
  ↓
v2.1: 完整情报系统 (新闻/巨鲸/链上 + Qwen分析)
```

## 🤝 贡献指南

### 更新版本文档流程

1. **发布新版本时**:
   ```bash
   # 1. 创建版本功能清单
   docs/10-版本更新/vX.X功能清单.md
   
   # 2. 更新CHANGELOG
   docs/10-版本更新/CHANGELOG.md
   
   # 3. 更新本README索引
   docs/10-版本更新/README.md
   
   # 4. Git提交
   git add docs/10-版本更新/
   git commit -m "docs: 添加vX.X版本文档"
   ```

2. **文档格式要求**:
   - 使用Markdown格式
   - 包含清晰的目录结构
   - 添加适当的emoji图标
   - 提供代码示例
   - 标注文件位置

3. **版本号规则**:
   - 主版本号(X): 重大架构变更
   - 次版本号(Y): 新功能添加
   - 修订号(Z): Bug修复
