## 💡 维护建议

### 日常开发规范

1. **临时报告文件**
   - 在开发过程中可以创建临时报告
   - 功能完成后及时归档到 `docs/99-归档/vX.X-开发记录/`
   - 不要在项目根目录长期保留临时文件

2. **版本更新流程**
   ```bash
   # 1. 更新VERSION文件
   echo "3.4.0" > VERSION
   
   # 2. 更新CHANGELOG
   vim docs/10-版本更新/CHANGELOG.md
   
   # 3. 创建版本发布文档
   vim docs/10-版本更新/v3.4.0_XXX功能发布.md
   
   # 4. 更新主README.md
   vim README.md
   
   # 5. 更新docs/README.md
   vim docs/README.md
   ```

3. **文档更新规范**
   - 新增重要功能，必须同步更新文档
   - 技术文档放在 `docs/03-技术架构/`
   - 使用指南放在 `docs/06-快速参考/`
   - 部署文档放在 `docs/07-部署运维/`

4. **归档规范**
   - 每个大版本创建独立归档目录
   - 归档目录必须包含README.md说明
   - 说明文档应包含：版本信息、主要功能、文档列表
