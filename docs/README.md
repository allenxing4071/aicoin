## 📝 文档贡献指南

### 文档命名规范

1. **顶级目录**: `XX-目录名称/` (XX 为两位数字编号)
2. **文档文件**: `XX-文档标题.md` (XX 为两位数字编号)
3. **编号规则**: 从 00 开始递增,不允许重复

### 文档编写规范

1. **Markdown 格式**: 所有文档使用 Markdown 格式
2. **标题层级**: 使用 `#` 表示标题,最多 6 级
3. **代码块**: 使用 ` ``` ` 包裹代码,指定语言
4. **链接**: 使用相对路径链接其他文档
5. **图片**: 存放在 `assets/` 目录

### 文档更新流程

1. 创建或修改文档
2. 更新对应目录的 README.md
3. 更新主 README.md 索引
4. 提交 Git 并注明变更内容
